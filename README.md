# Document Intelligence Platform — Skills-Only Workflow

Converting legacy documents and updating terminology is a bottleneck in every migration. This prototype shows how to automate it — using Claude Code skills, with no Python scripts.

## What this is

A three-stage workflow — **convert → rebrand → validate** — that takes a real Word document, converts it to DITA, applies a brand-terminology update, and runs a quality check on the result.

Only one file **`skill-creator`**, a meta-skill that generates other skills from a specification. The three pipeline skills — `convert`, `rebrand`, and `validate` — were each generated by passing a specification written in the **RTCCO framework** (Role, Task, Context, Constraints, Output) to `skill-creator`.

## Repo structure

.claude/commands/    ->   the skills (skill-creator, convert, rebrand, validate)

docs/rtcco-specs/    ->   the RTCCO specifications used to generate each skill

input/               ->   the source Word document

output/              ->   the converted and rebranded DITA files

validation/          ->   the validation report

## How to run

Each command is run one after another, in the Claude Code terminal:

/convert Sample_Doc_for_DITA_Conversion.docx
/rebrand Sample_Doc_for_DITA_Conversion
/validate Sample_Doc_for_DITA_Conversion

Each skill writes its own output to its folder automatically. No orchestrator — every stage is run and reviewed individually.

## The RTCCO framework

Every generated skill came from a specification with five parts:

- **Role** — the expertise the skill should have
- **Task** — what it does
- **Context** — background it needs every time it runs
- **Constraints** — hard rules it must follow
- **Output** — exactly where and how it saves its result

The specification files are in `docs/rtcco-specs/` — you can compare each one against the skill it generated in `.claude/commands/`.
